# Architecture Decision Records (ADR)

Этот каталог содержит **Architecture Decision Records** — краткие документы с важными архитектурными решениями проекта.

## Зачем
- фиксируем контекст и причины решения (почему так, а не иначе)
- упрощаем онбординг и ревью архитектуры
- уменьшаем риск "забытых" предположений

## Формат
Каждый ADR содержит:
- **Status**: Proposed | Accepted | Superseded | Deprecated
- **Context**: какие требования/ограничения привели к решению
- **Decision**: что именно принято
- **Consequences**: плюсы/минусы, что это меняет
- **Alternatives**: что рассматривали, почему отказались

## Нумерация и имена файлов
- `ADR-0001-<short-title>.md`
- номера не переиспользуем

## Изменение решений
- решение не редактируем "по-тихому" задним числом
- если нужно изменить — создаём новый ADR с новым номером, а старый помечаем **Superseded** и даём ссылку на новый

## Где искать архитектурные правила
- общий код/платформа: `internal/platform/*`
- сервисы: `services/*`
- контракты: `proto/*` (internal) и OpenAPI (позже)
