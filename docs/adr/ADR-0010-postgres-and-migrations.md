# ADR-0010: PostgreSQL and migrations strategy

- **Status**: Accepted
- **Date**: 2026-02-02

## Context
Нужна транзакционная БД для:
- пользователей/сессий
- подписок
- каталога
- пользовательских действий (progress, отзывы)

Важно иметь управляемую эволюцию схемы.

## Decision
- Используем **PostgreSQL** как основную транзакционную БД.
- Изменение схемы производится через **миграции** (инструмент будет выбран позже; например golang-migrate).
- Стратегия на старте: можно использовать один инстанс Postgres в dev, но логически разделять схемы/БД по сервисам.

## Consequences
### Positive
- зрелая БД, отличная экосистема
- транзакции, индексы, JSONB при необходимости

### Negative
- при росте потребуется шардирование/реплики/выделение отдельных кластеров

## Alternatives considered
- NoSQL как primary: усложняет транзакционные сценарии (billing, auth).
