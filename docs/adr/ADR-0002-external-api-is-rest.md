# ADR-0002: External API is REST

- **Status**: Accepted
- **Date**: 2026-02-02

## Context
Клиентам нужен стабильный внешний API. Требования:
- простая интеграция
- прозрачное кэширование на edge
- удобная отладка

## Decision
- Внешний API (BFF/Gateway) делаем **REST**.
- Версионирование путей: `/v1/...`.
- Единый формат ошибок (будет зафиксирован отдельным ADR/спеком API).

## Consequences
### Positive
- простота и предсказуемость
- хорошо работает с CDN/Reverse proxy
- проще мониторить и кешировать

### Negative
- при сложной агрегации данных возможен over-fetching/under-fetching

## Alternatives considered
- **GraphQL**: удобно для агрегации, но сложнее безопасность/кэширование и выше порог входа.
